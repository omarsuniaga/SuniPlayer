## Design Tactics

Apply these tactics to guide users and emphasize key actions:

### Color & Contrast
- Use primary color roles for primary actions to create high contrast
- Apply secondary/tertiary roles for supporting elements
- Dark on light (or light on dark) makes content easy to scan

### Hierarchy
- Emphasize importance using multiple cues: color, shape, text, and motion add clarity
- Make primary goals most prominent through size, color, placement
- Use lighter colors and subtle containers for supportive (non-required) actions
- Guide the eye from most to least important elements
- Place related items of a similar hierarchy next to each other

### Placement
- Position important actions at the top or bottom of the screen (reachable with shortcuts)
- Position primary actions at bottom for easy thumb reach
- Make primary action the final, most prominent element in vertical flow
- Avoid competing with other content for attention

### Shape
- Use rounded forms to reinforce distinct, touchable controls
- Apply expressive shapes (flower, sunny) for hero moments
- Keep shapes consistent within a flow for visual harmony

### Size
- Extra large for primary actions and hero moments
- Medium for secondary actions
- Custom-scale key numbers/data for scanning ease

### Spacing
- Generous spacing separates elements for clarity
- Even spacing between groups makes scanning easier
- Add space between shapes and data for comparison

### Typography
- Larger text for primary actions and key data
- Emphasized styles for important numbers
- Consistent text spacing maintains focus

### Motion
- Animations guide user attention and pace (e.g., breathing exercise)
- Use Material Spring Motion Tokens for natural feel
- Avoid multiple competing animations

### Grouping
- Neatly group related settings and options
- Use containment to separate primary from supporting content
- Uniform shapes and sizes create visual rhythm

## Best Practices

### Use Clear Scale and Placement

**Do:**
- Create one strong visual focal point
- Make supporting controls clear but less prominent
- Scale elements based on importance

**Don't:**
- Crowd the screen with equally prominent elements
- Make all elements large (creates unclear hierarchy)
- Leave settings ungrouped

### Reinforce with Consistent Color Roles

**Do:**
- Use primary color role for main actions
- Apply secondary colors for selected states and highlights
- Create visual hierarchy through color differentiation

**Don't:**
- Use the same color role for all actions and data
- Mix primary and secondary roles inconsistently
- Ignore the contrast between action and background

### Create Calm, Balanced Layouts

**Do:**
- Use uniform shapes and sizes for grouped content
- Add space between shapes for easy comparison
- Align elements in consistent flow

**Don't:**
- Overlap shapes or containers
- Mix different form types in same context
- Create visual chaos with competing animations

### Simplify through Structure

**Do:**
- Use clearly visible elements and sufficient contrast/size
- Create a clear hierarchy of importance
- Ensure key information is discernable at a glance
- Minimize steps in task flows

**Don't:**
- Increase UI complexity with unnecessary buttons, images, or text
- Force users to hunt for key information
- Overcomplicate primary task paths

## Testing & Iteration

Test designs with users to identify usability issues early.

**Compare versions:**
- Version 1 often has cluttered, competing elements
- Version 2 simplifies with clear hierarchy and containment
- Iteration moves from complex to calm

**Key improvements:**
- Group scattered options into organized lists
- Add containment to separate content types
- Reduce competing visual emphasis
- Apply consistent color roles throughout

## Case Study: Aura Breathing App

Aura demonstrates M3 Expressive usability tactics with four user goals:

1. **Start a breathing session** - Extra large button, high contrast, bottom placement
2. **Complete breathing session** - Animated flower shape guides pace, minimal competing controls
3. **View session results** - Flower-shaped containers for metrics, calm layout with spacing
4. **Check progress** - Primary data prominent, secondary accents for achievements

See [Applying M3 Expressive](https://m3.material.io/foundations/usability/applying-m-3-expressive) for detailed examples.
