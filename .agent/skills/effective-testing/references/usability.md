# Rapid Usability Testing

As detailed in Chapter 13, usability is a critical quality criterion. Rapid Usability Testing (RUT) is a quick way to assess the user experience without the need for a massive lab or months of planning.

## Understanding Users and Goals
- **Personas**: Create simple, realistic personas that represent your users (e.g., "Eric the Electrician").
- **User Goals**: What exactly is the user trying to achieve? (e.g., "Fix the wiring as fast as possible").
- **Living the Persona**: When testing, try to adopt the mindset, constraints, and "vibe" of the persona.

## RUT Basics
1. **Recruit Users**: Find 3-5 people who match your personas (internal users can work in a pinch).
2. **Plan and Prepare**: Define specific tasks for them to perform.
3. **Run the Test**: Observe them as they use the system. Don't lead them; let them struggle.
4. **Identify "Friction"**: Look for points where the user gets confused, frustrated, or slows down.

## Checking Usability
Use these guidewords and standards:
- **Nielsen-Norman Heuristics**: Visibility of system status, match between system and real world, user control and freedom, etc.
- **Accessibility (WCAG)**: Ensure the system is perceivable, operable, understandable, and robust for users with disabilities.
- **Learnability**: How fast can a new user become productive?

## The "I Am Testing an Off-the-Shelf System" Scenario
If you can't change the system, RUT still helps you:
- Identify training needs.
- Document workarounds.
- Advocate for better configurations.
